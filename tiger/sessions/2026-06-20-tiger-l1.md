# Session 2026-06-20 · Tiger L1 · 10 Characters × [[scan-assess]]

First Tiger run — establishes the **baseline dials** and the **predicted model frontier**. Method: 10 Characters from [[_roster]] each ran an L1 pass (brief: `_BRIEF.md`) over the one LLM call site. All findings code-grounded (`file:line`); the three highest-impact *new* findings were adversarially re-verified by the orchestrator before landing here. Per-Character reports: [[sam-staff-engineer]] · [[arjun-ml-platform]] · [[victor-finops-director]] · [[nadia-appsec-lead]] · [[tomas-prospective-buyer]] · [[elena-cto-founder]] · [[mariam-fintech-audit]] · [[diane-gov-onprem]] · [[mei-oss-maintainer]] · [[tania-scaleup-costcut]].

## Panel verdict: **fix-first, 10/10**
No Character said "not-yet" (the wrapping earns real trust) and none said "ship-as-is" (every one found a high-value engine gap). The convergence is striking: independent angles kept landing on the **same four issues**.

## Convergent themes (independent Characters → same finding)
1. **No prompt-caching** — Victor, Elena, Tania, (Mariam) all sharpest-finding'd this. `cache_control`/`cachePoint` = **0 hits repo-wide** (re-confirmed). The stable ~3-4K-token prefix (system + rubric + JSON schema) is re-billed at full input price on every distinct-commit scan. Quantified: ~30-90% off the input bill on a re-scan cadence; ~$155-450/yr on a 300-repo weekly fleet. **Zero quality risk** (output unchanged). The single biggest cost lever. → [[backlog#P0-1]]
2. **No prompt/response capture / no audit trail** — Sam, Nadia, Mariam. Only *failures* log (`scan.ts:281`); the prompt + raw output of a *usable* assessment are never persisted. Consequences span three Characters: can't debug a usable-but-wrong answer (Sam), can't detect a successful injection after the fact (Nadia), can't defend the number to a regulator or pull the prompt (Mariam) — and it **blocks Lens-C iteration** (no eval corpus to benchmark against). → [[backlog#P1-4]]
3. **Grounding the app already paid for never reaches the model** — the sharpest *new* class, two confirmed instances:
   - **Stack-fit computed, then withheld** (Arjun): `detectStackFit` recognizes `.ipynb`/ML and emits a caveat, but runs at `scan.ts:338` — *after* `assembleReport` (`:304`) and the `scoreInput` build (`:176`) — so it only lands in `warnings`. The model writes the roadmap + discrepancy audit **ML-blind on a web rubric.** "The right organ is built and not wired to the brain." → [[backlog#P0-2]]
   - **Priority files re-sorted alphabetically** (Sam): `pickFilesToFetch` signal-ranks files and GitHub quota fetches them, then `source.ts:454` re-sorts the array **alphabetically**, and the 22KB prompt window (`prompt.ts:90-93`) walks it in that order and breaks on overflow — so a large repo is judged on **alphabetical-luck** excerpts, and the discrepancy auditor is blind exactly where the smart-fetch was discarded. → [[backlog#P0-3]]
4. **The deterministic floor can ship as "AI" without the loud caveat** — Mei + Mariam + Diane, partially refined by Tomas:
   - On a **keyless** deploy (`LLM_PROVIDER=auto`, no key), `getProvider`→`geminiOrMock()` returns `MockProvider`, so `intendedProvider==="mock"` and `llmFailed` stays **false** (`scan.ts:118,279`) — the loud "AI unavailable" warning the *failure* path emits is bypassed for the keyless-*default* case (Mei). **Tomas refutes the stronger claim:** the *failure* degrade IS honestly labeled (warning banner + "Demo · deterministic rubric" chip + badge "· demo"). So the gap is narrower than "silent mock": disclosure exists via the chip, but not via the loud caveat, on the keyless-default route.
   - The **signed audit CSV has no engine column** (Mariam, Diane): a mock-degraded quarter is byte-identical to a model-scored one in the one artifact an auditor files. → [[backlog#P1-5]]

## Lens-C consensus (model optimization)
Unanimous shape: **`cheaper-holds` on the score · `mid-floor` on the roadmap + discrepancy audit.** Because scoring is guardbanded — configured ±25, but **realized ≈ ±15** after the 60/40 blend (`engine.ts`) — the model barely moves the number, so the score is nearly model-insensitive (cheap-safe). The unbounded value is the **roadmap** and the **discrepancy audit** (reasoning over excerpts), for which **sonnet is the floor** and **opus/thinking helps only the discrepancy audit on complex repos**. Concrete optimizations surfaced: a **two-model split** (cheap for scoring/summaries, strong for roadmap+discrepancies — Tania), a **scoped thinking-budget knob** (none exists; `config.ts` has only temperature/maxTokens — Arjun), and **right-sizing the public default** (gemini-3-flash may already hold the floor — Mei/Tomas, but Tomas warns the buyer judges the product on that cheap scan). All of this needs the live `benchmark` pass to confirm. → [[backlog#P2]]

## Strengths to protect (as decision-useful as the gaps)
- **Honest billing** — token usage committed *only on a usable attempt* (`scan.ts:206-219`); a failed attempt's tokens are dropped. Every cost-savvy Character credited it.
- **The wrapping stack** — retry → failover → flagged mock-degrade, per-call timeout + total budget, abort, schema-forced output + never-throw validate + coverage gate, field/array caps. Nadia: "genuinely careful." Arjun: "best plumbing I've seen on a tool like this."
- **claude-cli secret hygiene** — deletes `ANTHROPIC_API_KEY` from the child env + model-id regex-gate before a `shell:true` spawn (`claude-cli.ts`). Nadia: senior-grade.
- **Tamper-evidence** — per-row HMAC (`audit-integrity.ts:58-71`) + CSV content-hash header (`history/route.ts:115`). Mariam (undersold by the engine note).
- **Deployability for constrained buyers** — GHES base-URL knobs (`github/host.ts`) + OpenAI base-URL override now exist and are wired; Diane's air-gap blocker from the prior UAT pricing run is **refuted**.

## What this run taught the skill (→ finetune)
- **Grounding has two directions.** "Does context reach the prompt?" (in) AND "does the model's provenance survive into the durable artifact?" (out). The richest findings were *out*-direction (audit CSV) and *plumbing* (computed-but-not-wired).
- **The sharpest grounding finding is "computed-but-not-wired".** The app already paid for context (a stack detector, a ranked fetch) and then dropped it before the prompt. Auditing "does context the app ALREADY HAS reach the model?" beats "does it exist?". Folded into [[business-value]] + the skill.
- **Degrade-path disclosure parity** is a finding class (failure-route discloses loudly; keyless-default route doesn't). Folded into [[engine-quality]] + the skill.
- **Configured ≠ realized** — a clamp+blend means the *realized* guardband (±15) ≠ the configured constant (±25). The skill now tells Characters to compute the realized effect.
- **Two-model split** named as a Lens-C pattern.
