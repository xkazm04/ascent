## Suggested fix-wave split

Seven sessionable waves, ordered safest-and-highest-value first. Each shares one mental model so fixes compound. Every wave: atomic commit per fix → `tsc` clean → full `vitest` green (baseline 2585) before the wave summary.

- **Wave 1 — Dead code removal (Theme E, ~7).** Pure subtraction, all grep-verified zero-caller. Delete `ScanGallery`, `ReportTabBar` (relocate the `ReportTab` type), `getOrgContributors`+type, `unwatchReposForInstallation`, `simulateOrgFix`, `updateRecommendationStatus`, `weightTint`/`weightText`+Flight-Deck leftovers, and the `@/lib/db`→forecast re-export. Update the tests that exist only to exercise the deleted code.
- **Wave 2 — Security/safety primitives (Theme A, ~6).** Extract `src/lib/export/csv.ts` (canonical escaper + filename sanitizer) and replace the 4+8 copies — closes the `org/repositories` formula-injection drift. Unify the SSRF host guard, route `members`/`invites` through the shared `getOrgId` (fixes slug-normalization drift), and extract one `parseRepo`. *(Tenant-read IDOR gate held for explicit sign-off — see below.)*
- **Wave 3 — Shared infra plumbing (Theme C, ~6).** A `src/lib/sse-server.ts` `send`+headers helper; point onboarding's `importScan` at the tested `readSSE`; a shared GitHub `ghHeaders()`; route `retention.ts` through `db/client`'s `withRetry`; collapse the LLM timeout/abort wiring + `LLM_TIMEOUT_MS` onto `config.ts`.
- **Wave 4 — Scoring/domain logic (Theme D, ~5).** One gate evaluator behind both `evaluateGate`/`evaluateGateLite` signatures; `levelIndex()`/`nextLevel()` helpers in `model.ts`; a shared `isDimId`; extract the apply-PR pipeline; single-source the CI-snippet preamble.
- **Wave 5 — UI shared constants & chips (Theme F-1, ~6).** Import `STATUS_LABEL`/`STATUS_ACCENT`, `POSTURE_ORDER`, `SCHEDULES` from their canonical homes; a `DIRECTION_TONE` helper; reuse `DeltaTag`; compose `Kicker` in `Dateline`; import `humanizeDays`.
- **Wave 6 — UI shared components & markup (Theme F-2, ~6).** Shared `src/lib/og` brand chrome; a `ScopeFilterBar` component; extracted sign-in button chrome; shared `DimRow` + prev-period grid; a `LiveRepoSeed` mapper; a shared Remotion `Metric`; a `bulkTagRepos()` client helper.
- **Wave 7 — Medium/Low cleanup tail.** The remaining 63 Medium + 53 Low (stale comments, `import type` fixes, write-only fields, console/debug, env-flag `envBool()` helper), batched by file as time permits.

**Held for explicit sign-off (behavior-affecting, not pure refactor):** the tenant-read IDOR gate (`usage` page + route → `canReadOrg`). The hand-rolled copies are *missing* branches `canReadOrg` enforces (Supabase login-wall, `openOrgDashboardsEnabled`), so unifying onto it **changes the access decision** (a correct hardening, but a behavior change). Flagging per the skill's "don't touch auth unless explicitly asked" rule.
