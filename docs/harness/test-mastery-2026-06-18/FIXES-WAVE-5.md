## What was fixed (the invariant each test now pins)

1. **`mergeStars`.** An unchanged star keeps its **same object identity** (`===`) across a merge (so React doesn't re-animate it); a changed field swaps to the fresh object; no duplicate ids; prev order preserved; removed star drops, new star appends.
2. **`canRunReal` money gate.** Missing install/credit → false; a **stale credit for another tenant** (`credit.org !== sourceLabel`) → false (cannot bill the wrong tenant); matching org with `unlimited`/`balance>0` → true; `balance 0` → false. The exact boolean expression was moved verbatim.
3. **Optimistic watch/schedule rollback.** An optimistic flip updates the target and leaves siblings identical; a non-2xx response **restores the EXACT prior** `watched`/`scanSchedule` (no success-theater); an absent id is a no-op. Only the pure next-state transform was extracted; fetch/setState/control-flow stayed in the component.
4. **SSE import parser.** A well-formed multi-event stream fires events in order; CRLF tolerated; an event split across two `read()` chunks is buffered and parsed once; a malformed/truncated chunk throws nothing and emits no bogus event; abort → `{aborted:true}`; no chunk within `STALL_MS` (fake timers) → `{aborted:true, stalled:true}`; the watch/schedule/mock cost-disclosure request body is pinned.

## Patterns established (catalogue items 24–28)

24. **Behavior-preserving extraction for testability.** To test logic trapped in a `"use client"` component (no jsdom), move the pure function **verbatim** into a plain `.ts` module and re-import it; verify with a targeted `tsc --noEmit | grep <files>` plus `next build` "compiled successfully" to confirm the use-client boundary survived. *(mergeStars, canRunReal, watchState)*
25. **Identity-preservation assertion.** For a merge/reconcile used in live refresh, assert unchanged items keep their **same object reference** (`===`) so the UI doesn't re-animate, and that changed items are swapped. *(mergeStars)*
26. **Optimistic-rollback exactness.** Assert a non-2xx response restores the **exact** prior value (not just "some" rollback), so success-theater (UI shows saved, server didn't) is impossible. *(watchState)*
27. **Abort-aware fake stream.** To test stream stall/abort deterministically, use a `ReadableStream` that errors itself on the abort signal (mirroring how `fetch` cancels its body) + fake timers, so the test can't hang. *(importScan)*
28. **Test-in-place over extraction when already a module.** If the "frontend" logic is already a plain `.ts` consuming a mockable stream/fetch, drive it with a fake rather than refactoring — lower risk, zero source change. *(importScan)*
